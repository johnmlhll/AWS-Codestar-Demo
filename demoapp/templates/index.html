Welcome to this Infrastructure Demo of a CI/CD pipline from local host to AWS cloud service. The deployment and pipeline was done via AWS CodeStar using AWS Elastic Beanstalk (managed service) on top of an EC2 instance for the deployment. This webapp was developed in Python (on Django v3.2.6 framework) and was truly fun to do; yet purposely simple as this project is about showcasing the AWS codestar CICD pipeline. CICD Updating Experience

The initial deployments went smoothly updating v1.0.0 from local pushes to the Github Repo, which in turn triggered from master branch the Codestar pipeline process as follows:

  1. Git Hub Master Branch change
  2. AWS CodeBuild Execution
  3. AWS Cloudformation Build Execution
  4. AWS EBS Deployment Change Creation
  5. AWS EBS Deployment Execution into target EC2 instance

CodeStar GUI in action looked like this:

As you can see, the version upgrade to v1.0.1 from v1.0.0 was successful proving the CICD flow in action using Codestar as a platform. I pushed to master direct and also via PR from development_master after locking down master branch for protection; given that it is linked to my Codestar project. Once the Pull Request was approved and merged into master, the CICD run was immediately triggered, connecting my Github repo to the Codestar CICD process. As some point in the future, I will replicate the same code (there or there abouts) in a new CICD project on AWS using Jenkins into CodeDeploy/EC2 and compare the results in effort and cost. For this project, I did not buy a domain name on cost basis and use AWS Certificate Manager to issue an in house SSL cert for it. This is why the endpoint http://ec2-63-33-220-22.eu-west-1.compute.amazonaws.com/ works only in http, not https. I also did not see the point to use a single instance (workaround) proxy 443 to 443 endpoint as it was upon reflection a distraction from the CICD nature of this project.
